# Secrets Usage

Source page version: 10

## Prerequisites

- Access to Azure Key Vault
- Appropriate permissions to use GitHub Actions and manage secrets
- Team-specific Key Vault configured (e.g., prod-github-<team>-kv)
- Variables.yml and repos.yml configuration files

## Step-by-step instructions

1. Refer directly to Azure Key Vault secrets in GitHub workflows:
   - Use the Azure CLI action in your workflow to retrieve a secret:
     ```yaml
     - name: get-secret
       uses: azure/cli@v2
       with:
         azcliversion: latest
         inlineScript: |
           secret=$(az keyvault secret show --name <secret-name> --vault-name <kv-name> --query value -o tsv)
           echo "::add-mask::$secret"
           echo "ApiKey=$secret" >> "$GITHUB_OUTPUT"
     ```
   - Use the secret value in subsequent steps:
     ```yaml
     - name: set-app-settings
       uses: microsoft/variable-substitution@v1
       with:
         files: 'appsettings.json'
       env:
         ApiKey: ${{ steps.get-secret.outputs.ApiKey }}
     ```

2. If you must use GitHub Secrets sourced from Azure Key Vault:
   - Add secrets to the team Key Vault matching the team and variables.yml configuration:
     ```yaml
     secrets_kv_rg: prod-github-teams-rg
     secrets_kv_name: prod-github-<team>-kv
     ```
   - In the repos.yml file, reference Key Vault secret names for GitHub secrets:
     ```yaml
     secrets:
       <github-secret-name>:
         az_secret_name: <kv-secret-name>
     ```
   - Run pipelines to populate GitHub secrets; secrets will be updated on pipeline creation or execution.

3. To update a secret on demand:
   - Run the GitHub Apply pipeline with stage set to `ApplyTeam` and `name` set to your team name.

4. For large team/repo deployments:
   - Use the target variable to update a specific secret faster:
     ```
     module.repo["<repo-name>"].github_actions_secret.secret["<gh-secret-name>"]
     ```

## Warnings or Important Notes

- Avoid copying secrets from Azure Key Vault to GitHub. Prefer direct referencing instead.
- Ensure Key Vault names and variables configurations match your team and repo settings.

## Key Concepts

- **Azure Key Vault:** A cloud service for securely storing and accessing secrets, keys, and certificates.
- **GitHub Secrets:** Encrypted environment variables used in GitHub Actions workflows.
- **Azure CLI Action:** GitHub Action allowing the execution of Azure CLI commands within workflows.
- **Variables.yml and repos.yml:** Configuration files specifying team and repo-specific secret settings and environment configurations.
- **GitHub Apply Pipeline:** Pipeline used to update secrets and variables in GitHub repositories.
