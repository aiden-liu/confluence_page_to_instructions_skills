It is a historized copy of the source table.
This is the equivalent of the operation data store (ODS) tables from the Enterprise Data Warehouse (EDW). Changes are fully tracked, so this performs like an SCD 2 table.
Like bronze objects, copper objects are stored in a copper schema that is related to the source.
Where possible, use Lakeflow Declarative Pipelines for the copper layer.
This allows for consistency, and follows Databricks recommended processes.
